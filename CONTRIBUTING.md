# Contributing Guidelines

Thank you for your interest in this project! 

> **Notice:** This project is classified as **Proprietary and Closed Source**. 

## External Contributions

Because this is an internal or proprietary commercial project, we do not accept unsolicited Pull Requests from external developers to protect our intellectual property and client confidentiality.

## Bug Reports and Feature Requests

If you are an authorized user, contractor, or auditor:
1. **Issues**: You may submit bug reports or feature requests via the Issue Tracker. 
2. **Security Vulnerabilities**: If you discover a security vulnerability, please do NOT report it in the public issue tracker. Contact the repository owner directly.

## Internal Organization Contributions

If you are an internal team member or authorized contributor:
1. Please coordinate with the lead maintainer before starting work on a new feature or major refactoring.
2. Ensure you are working from the latest `main` or development branch.
3. Submit a Pull Request for review with a clear description of the business value and systems impacted.
